# 演示视频讲稿

大家好，我们小组完成的项目是“基于 YOLO 的校园车辆与车牌检测系统”。

首先介绍项目背景。校园门口、停车场和道路区域经常需要记录车辆通行情况，如果完全依靠人工查看图片或监控画面，效率比较低，也不方便后续统计。因此我们希望利用目标检测技术，自动定位图片中的车辆和车牌区域，并把检测结果可视化展示出来。

本项目主要完成了四个部分。第一是数据集整理，我们合并了 Zenodo、Roboflow 和 Kaggle 三个公开车牌检测数据来源，最终形成 25554 张图片的数据集。第二是模型训练，我们使用 PyTorch 实现了一个轻量 YOLO 风格检测模型，输入尺寸为 256×256，采用 10×10 网格预测目标置信度、边界框和类别。第三是系统实现，我们开发了 Flask 非静态版，可以上传图片并调用训练好的模型进行检测。第四是演示保障，我们还提供了一个静态 HTML 版本，即使不启动后端服务，也可以完成页面展示和结果图导出。

下面展示系统功能。打开非静态版页面后，用户可以上传一张车辆图片。点击开始检测后，系统会输出带检测框的图片，并在右侧显示车辆数量、车牌数量、平均置信度和目标坐标。同时，系统会把检测到的车牌区域裁剪出来，方便进一步处理或写入报告。

训练结果方面，我们使用 22094 张图片作为训练集，2310 张图片作为验证集，1150 张图片作为测试集。模型训练 12 轮，最佳验证损失为 0.1420，测试集损失为 0.1196。相比最初只使用小数据集训练的版本，新增数据后测试集表现更稳定，也更能体现模型训练工作量。

项目亮点主要有三个。第一，完成了从数据集下载、合并、训练到应用展示的完整闭环。第二，提供非静态版和静态版两套演示方式，降低了答辩现场的运行风险。第三，系统保留了 YOLOv5 ONNX 接口，后续可以继续替换为更强的 YOLOv5 或 YOLOv8 模型。

当然，项目还有可以改进的地方。目前模型重点偏向车牌区域检测，车辆整体检测主要由 OpenCV 候选区域进行补充。后续可以增加更多校园真实场景数据，训练完整车辆与车牌联合检测模型，并进一步接入车牌字符识别和实时视频检测功能。

以上就是我们项目的主要演示，谢谢大家。
